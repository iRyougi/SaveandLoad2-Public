# SaveandLoad2 v0.2.0 (Pre-release)

> 发布日期：2026-05-22
> 类型：预发行版本（Pre-release）
> 上一版本：[v0.1.0](./RELEASE_v0.1.0.md)

本版本最大变化是引入 **Pro 模式（专业模式）**：通过可选安装的游戏 Mod（`SnL2.EliteLogger`）实时记录战斗过程，把原本 `.run` 文件里看不到的逐回合战斗细节也纳入复盘范围。

---

## 重点更新

### 全新「Pro 模式」

`.run` 存档只记录"这局打完是什么样"，但不会告诉你"每回合具体抽了什么、打了什么、被打了多少"。Pro 模式通过随软件附带的 Harmony Mod 把战斗过程实时写盘，让复盘真正深入到回合级别。

- **一键安装/卸载**：设置页提供 Pro 模式卡片，自动定位 Steam 中的《杀戮尖塔 2》安装路径并探测 `bundle.pck` / `game.pck` / `baselib`，然后部署 `SnL2.EliteLogger` Mod。
- **状态机管理**：`ProModeService` 内部使用状态机，覆盖"未启用 / 已安装 / 升级中 / 出错"等场景，避免反复安装造成冲突。Mod 的 `VERSION.txt` 与软件本体版本号保持一致，便于判断需不需要升级。
- **全战斗捕获（schema 2.0.0）**：早期版本仅记录精英战，新版本扩展到所有战斗——普通怪、精英、Boss 全部纳入。每场战斗都包含：抽牌、打牌、伤害、护甲、能力（Power）变化等逐回合数据。
- **存储位置带兜底**：Mod 同时写 Steam Cloud 路径与本地 AppData 路径；当 Steam Cloud 未同步 `combat_logs` 时，Reader 自动回落到 AppData 读取，避免数据丢失。
- **按 `run_id` 分组**：战斗日志按对局分组保存，便于与 `.run` 文件一一对应。
- **手动清理**：设置页支持手动清理 7 天前的战斗日志，避免占用过多磁盘空间。

> 持久化范围：卸载 Mod 不会删除已生成的战斗日志；想要彻底清除需手动清理。即便软件本体已关闭，Mod 仍会在游戏运行时持续记录。

### AI 复盘接入战斗数据

- AI 复盘的提示词现在会自动注入精英/Boss 战斗摘要，让 AI 不再只看到"获得了什么卡"，而能看到"每回合具体做了什么"。
- 提示词上下文更完整，复盘质量显著提升。

### 对局详情页增强

- 出现过精英战的楼层会显示徽章标记。
- 点开徽章可以查看该场战斗的逐回合时间线（抽牌、打牌、伤害、Power 变化）。
- 战斗日志数据通过新的 view-model 接入，与原有楼层时间线无缝衔接。

### 其他改进

- Mod 项目在 Visual Studio 解决方案中可见，便于开发者审阅源码。
- 战斗日志 JSON Schema 冻结为 v1.0.0，并附带样例数据，方便第三方工具消费。

## 已知问题与限制

- Pro 模式 Mod 仅支持当前版本《杀戮尖塔 2》。游戏更新后 Harmony 补丁可能失效，需等待软件升级。
- 战斗日志 schema 2.0.0 与早期版本不兼容；如果存在旧日志，建议清理后重新生成。
- 安装 Mod 需要软件具备访问游戏目录的权限，部分用户可能需要以管理员身份运行。
- 联机存档下 Pro 模式数据捕获仍以本地玩家为准，远端玩家行为不会写入日志。
- 目前仅打包 Windows 版本。

---

## 升级注意事项

- 从 v0.1.0 升级到 v0.2.0：
  1. 安装新版本软件本体。
  2. 如希望体验 Pro 模式，到「设置 → Pro 模式」点击启用，软件会自动部署 Mod。
  3. 启动游戏开始一局新对局，对局结束后即可在详情页看到战斗时间线。
- 不开启 Pro 模式的话，软件行为与 v0.1.0 完全一致。
