# SaveandLoad2 v0.3.2 (Pre-release)

> 发布日期：2026-06-11
> 类型：预发行版本（Pre-release）
> 上一版本：[v0.3.1](./RELEASE_v0.3.1.md)

本版本是一次**安全加固 + 流畅度优化**的维护更新，没有新功能。重点解决三类问题：修复已知依赖漏洞与外链安全隐患、防止设置文件在意外断电/崩溃时整体丢失、消除战斗记录页点选对局时的卡顿。建议所有用户更新。

---

## 重点更新

### 流畅度：点选对局不再卡顿

战斗记录页**每次点选对局时 100-260ms 的界面冻结已基本消除**（实测最差从 260ms 降至 80ms 以内，多数点击完全无卡顿）：

- 选中对局后，概况栏、卡组、AI 复盘卡片**立即显示**，逐层时间轴在约 0.2 秒内流式补全，期间界面始终可响应。
- 顺带修复了一个内存问题：反复点选对局时，旧的详情数据不再滞留内存（此前长时间使用内存会持续增长）。

### 安全：外链白名单与依赖漏洞修复

- **更新公告与新闻里的链接**现在只允许打开 https 协议下的可信域名（GitHub 官方域及国内镜像）。如果远端数据被篡改塞入异常链接，应用会提示"链接不安全，已阻止打开"，不会再交给系统执行。
- **Steam 头像**只接受 http/https 来源，**自定义档案端点**保存时会校验格式（需 `http(s)://` 开头且含 `{steamid}` 占位符），杜绝本地文件协议被滥用的可能。
- 升级了数据采集 mod 的 `System.Text.Json` 依赖，修复两个已公开的高危漏洞（GHSA-hh2w-p6rv-4g7w / GHSA-8g4q-xg66-9fp4）。

### 可靠性：设置不再"一夜回到解放前"

修复了一个可能导致**全部设置丢失**的隐患：此前保存设置时如果恰好碰上崩溃或断电，`settings.json` 可能只写了一半，下次启动会静默回退到默认状态（SteamID、API Key、目录配置全部消失）。

- 现在采用**原子写入**：任何时刻磁盘上要么是旧的完整配置、要么是新的完整配置，不会出现半截文件。
- 如果检测到设置文件已损坏，会先把损坏文件**备份为 `settings.json.corrupt`** 再回退默认——你的配置仍可手工找回，不会被悄悄覆盖销毁。

## 升级注意事项

- 从 v0.3.1 升级：直接覆盖安装即可，所有设置与数据保留。
- 本版无功能变化、无设置项变化，纯修复更新。
- 若更新公告/新闻里出现"链接不安全，已阻止打开"的提示，说明白名单拦截了非预期链接，属于预期保护行为，欢迎反馈具体场景。
